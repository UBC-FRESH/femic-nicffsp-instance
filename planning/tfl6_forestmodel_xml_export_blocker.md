# TFL 6 ForestModel XML Export Blocker

## Purpose

This note records the first P4.2 ForestModel/XML export attempt and the
blocking schema mismatch it exposed.

Governing issue: `#37`.

The attempted command was:

```powershell
python -m femic export patchworks `
  --instance-root . `
  --tsa tfl6 `
  --bundle-dir data\model_input_bundle `
  --checkpoint data\model_input_bundle\input_geometry\aflb_current.feather `
  --output-dir output\patchworks_tfl6_validated `
  --start-year 2026 `
  --horizon-years 300 `
  --ifm-mode proportional
```

The command failed before writing a usable ForestModel XML package:

```text
Patchworks export failed: au_table.csv missing required column: tsa
```

No ForestModel XML, fragments shapefile, Matrix Builder output, Patchworks
runtime package, or publication artifact should be treated as generated by this
attempt.

## Root Cause

The P4.1 reviewed TFL 6 bundle uses the current TFL6-facing audit schema:

- string AU keys such as `cwhvm1_hw_cw_l`;
- string curve IDs such as `nat_cwhvm1_hw_cw_l` and
  `tipsy_future_managed_cwhvm1_hw_cw_l`;
- curve point columns `age` and `volume_m3_per_ha`; and
- explicit audit columns such as `canonical_curve_au_id`,
  `natural_curve_id`, `treated_curve_id`, `curve_lane`, and
  `curve_family`.

The current generic FEMIC Patchworks exporter still expects the legacy
FMG bundle schema:

- `au_table.csv` must include `tsa`;
- `au_id`, `managed_curve_id`, and `unmanaged_curve_id` are coerced to
  integers;
- `si_level` is expected rather than `si_class`;
- `curve_points_table.csv` must include `curve_id`, `x`, and `y`; and
- `curve_table.csv` optionally uses `curve_type`.

Therefore adding only a `tsa` column would not be sufficient. P4.2 needs a
reviewed schema bridge before XML generation can proceed.

## Required Repair

The next bounded P4.2 repair should create one of these explicit bridges:

1. **Preferred first repair:** generate an exporter-compatible view of the
   reviewed TFL6 bundle under generated output space, with deterministic numeric
   AU and curve IDs plus a tracked mapping/audit note back to the reviewed TFL6
   string IDs.
2. **Broader code repair:** teach the generic FMG Patchworks exporter to accept
   string AU and curve IDs directly, if Patchworks XML/fragments semantics can
   support that without breaking existing K3Z/TSA29/MKRF flows.

The first repair is lower risk for Phase 4 because it does not change existing
cross-instance exporter behavior. It should preserve:

- AFLB as the Patchworks stand universe;
- THLB/NTHLB as managed/unmanaged share state;
- non-null natural and treated curve mappings for every stand;
- sparse treated-curve fallback audit fields;
- deferred `unassigned_review_required` harvest-system state; and
- blocked final clearcut-and-plant eligibility until reviewed
  ground/cable/heli assignment exists.

## Handoff

P4.2 remains open. The next executable task is a targeted schema-bridge repair,
not Matrix Builder or runtime packaging.
